---
title: Install SEO skills that teach agents how to use reports
description: Add focused SEO skills to Codex, Claude Code, or a project so agents can choose the right report, read its evidence, and verify each change.
---

An SEO skill is a focused instruction file for an agent. It explains the
technical question a report answers, when the evidence is useful, which command
or MCP tool to call, how to read partial data, and what to verify after a fix.

The skill does not contain a second report implementation. It teaches the agent
to call SEO Skills CLI. The skills and runtime are designed to be installed
together: the `seo` package does the work, while the skills teach the agent how
to use it well.

## Install the collection with the standard skills installer

Install the repository collection with the command used across the agent
skills ecosystem:

```sh
npx skills add iannuttall/seo
```

The installer handles the supported agent destinations. The repository remains
the skill source, while report execution comes from the local `seo` package.

For the recommended setup, install both parts:

```sh
npm i -g seo
npx skills add iannuttall/seo
```

## Inspect the copies inside the SEO package

The published package includes the same skill files so the CLI, MCP server,
and release checks can inspect them locally:

```sh
seo skills list
seo skills path quick-wins
seo skills path
```

## Skills teach the part a schema cannot

`seo reports describe quick-wins --json` can tell an agent which parameters
are valid. It cannot explain why positions 4 to 10 are a useful review set, why
CTR should be compared in context, or why the retained rows are not the whole
Search Console universe.

A report skill fills that gap. Each focused skill should teach the agent to:

- discover and describe the report before running it;
- start with compact output and request a smaller follow-up;
- keep observations separate from heuristics and actions;
- inspect date ranges, row caps, partial states, and caveats;
- propose a bounded fix with a verification step;
- avoid traffic, ranking, indexation, and AI citation promises.

That last point matters. A tool can find a title conflict or an underperforming
query. It cannot promise what a search system will do after the edit.

## Use skills and MCP together

The [local MCP server](/docs/mcp) gives an agent three compact tools for report
discovery and execution. Skills tell the agent which report fits the request
and how to interpret the result.

Without MCP, the same skill can call:

```sh
seo reports list --json
seo reports describe quick-wins --json
seo reports run quick-wins --params '{"site":"sc-domain:example.com"}' --json
```

Both routes share the same report registry and core analysis.

## Discover canonical skills from the site

Compatible tooling can read the
<a href="/.well-known/agent-skills/index.json" target="_blank" rel="noreferrer">machine-readable SEO skill catalog</a>.
Each entry points to the canonical `SKILL.md` file and includes a SHA-256 digest
for content verification.

The catalog is useful for discovery. The `seo` npm package remains the runtime
an agent needs to execute reports locally.

## Update installed copies deliberately

Skills are copied files, so they do not update themselves when the repository
changes. Review local edits before running the standard installer again:

```sh
npx skills add iannuttall/seo
```

Project-specific instructions may be worth keeping. A blind overwrite is a
poor update strategy when someone has taught the skill about a release process,
CMS, or definition used only by that project.

The [agent workflow guide](/docs/agents) covers report selection and evidence
handling after installation.
